# TG Relay

A personal Telegram relay served as a GitHub Page. Private, password-locked, Material You design.

## Architecture

```
GitHub Actions (cron / manual)
  └─ fetch_messages.py  ──→  data/chats.json  ──→  GitHub Pages (index.html)
                                                         │
                                          User queues actions in UI
                                                         │
                                          Dispatch btn → process_queue.yml
                                               └─ process_queue.py → Telegram
```

## Setup

### 1. Generate a Telegram session file

```bash
pip install telethon
python -c "
from telethon.sync import TelegramClient
client = TelegramClient('session', YOUR_API_ID, 'YOUR_API_HASH')
client.start()
print('Session ready: session.session')
"
```

Get `API_ID` and `API_HASH` from https://my.telegram.org

### 2. Convert session to base64

```bash
base64 -w 0 session.session > session.b64
cat session.b64   # copy this value
```

### 3. Add GitHub Secrets

Go to **Settings → Secrets and variables → Actions → New repository secret**:

| Secret name | Value |
|---|---|
| `TG_SESSION_B64` | output of the base64 command above |
| `TG_API_ID` | your Telegram API ID (number) |
| `TG_API_HASH` | your Telegram API hash |
| `DATA_ENCRYPTION_KEY` | any long random passphrase |

### 4. Set your password

The default password is `relay123`. To change it:

```js
// In a browser console or Node:
const hash = async p => [...new Uint8Array(
  await crypto.subtle.digest('SHA-256', new TextEncoder().encode(p))
)].map(b=>b.toString(16).padStart(2,'0')).join('')

hash('your_new_password').then(console.log)
```

Copy the resulting hex string into `index.html` at:
```js
passwordHash: "paste_here",
```

### 5. Configure repo details in index.html

```js
const CONFIG = {
  passwordHash: "...",       // from step 4
  githubOwner:  "your_username",
  githubRepo:   "your_repo_name",
  githubToken:  "",          // see step 6
};
```

### 6. GitHub Personal Access Token (for triggering workflows from the UI)

1. Go to https://github.com/settings/tokens
2. Generate a **classic** token with scopes: `repo`, `workflow`
3. **Do not commit this token.** The UI will prompt you to paste it into `CONFIG.githubToken` (stored in memory only, cleared on refresh). For a more permanent solution, store it in `sessionStorage` after unlock.

### 7. Enable GitHub Pages

- **Settings → Pages → Source: Deploy from a branch → `main` / `/ (root)`**

### 8. Run the first fetch manually

- **Actions → Fetch Telegram Messages → Run workflow → Force full fetch: true**

## Data files (generated by Actions)

| File | Description |
|---|---|
| `data/meta.json` | Sync metadata, last run info |
| `data/chats.json` | Plain JSON chat/message dump (served to Pages) |
| `data/chats.enc` | Fernet-encrypted version (for Python pipeline) |
| `data/queue_results.json` | Results of last queue dispatch |

> **Note:** `chats.json` is unencrypted so the browser can read it. For sensitive use, implement client-side Fernet decryption using the `data/chats.enc` file and pass your key in the password dialog.

## Queue action types

| Type | Required fields |
|---|---|
| `send_message` | `chat_id`, `text`, optionally `reply_to` |
| `react` | `chat_id`, `msg_id`, `emoji` |
| `mark_read` | `chat_id` |
| `download_request` | `chat_id`, `msg_id` |
| `pin_message` | `chat_id`, `msg_id` |
| `forward` | `from_chat_id`, `msg_id`, `to_chat_id` |

TG_API_ID              Telegram API ID
